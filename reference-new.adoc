[role="lead"]
您现在可以部署和升级到 Cloud Volumes ONTAP 9.18.1。您可以在 Console 中查看此版本的基本版本以及可用于部署和升级的最新补丁版本。

要了解最新版本的 NetApp Console 中引入的 Cloud Volumes ONTAP 功能，请查看 https://docs.netapp.com/us-en/storage-management-cloud-volumes-ontap/whats-new.html["Cloud Volumes ONTAP"^]中的新增功能。



== 9.18.1 GA (2026 年 2 月 10 日)

Cloud Volumes ONTAP 9.18.1 General Availability 版本现在可用于在 Amazon Web Services (AWS) 和 Microsoft Azure 中进行部署和升级。此版本无法在 Google Cloud 中进行部署和升级。



== 升级说明

通读这些注释、了解有关升级到此版本的更多信息。



=== 如何升级

Cloud Volumes ONTAP的升级必须通过控制台完成。您不应使用系统管理器或 CLI 升级Cloud Volumes ONTAP 。这样做会影响系统稳定性。

link:http://docs.netapp.com/us-en/bluexp-cloud-volumes-ontap/task-updating-ontap-cloud.html["了解控制台通知您时如何升级"^] 。



=== 支持的升级路径

您可以从 AWS 和 Azure 中的 9.17.1 P1 及更高版本修补程序升级到 Cloud Volumes ONTAP 9.18.1。Console 将提示您将符合条件的 Cloud Volumes ONTAP 系统升级到此版本。


NOTE: Cloud Volumes ONTAP 9.18.1 无法在 Google Cloud 中升级。



=== 停机

* 升级单节点系统会使系统脱机长达 25 分钟，在此期间 I/O 会中断。
* 升级HA对不会造成中断、I/O不会中断。在此无中断升级过程中、每个节点会同步升级、以便继续为客户端提供I/O。
